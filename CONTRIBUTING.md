# Contributing

Thanks for helping improve Marwa DB.

## Before You Start

- Read the [Code of Conduct](CODE_OF_CONDUCT.md).
- Check the existing [README.md](README.md) for setup, commands, and project conventions.
- Keep changes small, focused, and backward compatible unless a breaking change is explicitly requested.

## Development Setup

- Install dependencies with `composer install`.
- Run the test suite with `composer test`.
- Run the full CI gate with `composer run ci`.

## Workflow

- Prefer editing existing code instead of creating duplicate implementations.
- Follow the existing code style and architecture.
- Add or update tests for any behavior change.
- Keep public API changes minimal and document them in the README when needed.

## Pull Requests

- Describe what changed and why.
- Mention any behavioral changes or compatibility notes.
- Include test results when relevant.

## Reporting Issues

- Include the PHP version, database driver, and steps to reproduce.
- Add relevant error messages, SQL, or stack traces when available.
